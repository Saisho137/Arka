# Arka — Reutilización y Versionado

**REGLA:** Patrones transversales ya implementados en `ms-inventory` se DEBEN copiar-y-adaptar, cambiando solo lo específico del dominio (entidades, tópicos, payloads, `MS_SOURCE`). Desviaciones permitidas solo por diferencias de tecnología (MongoDB vs PostgreSQL) o lógica de negocio única.

## Componentes Reutilizables

| #   | Componente                                                                                                                                                                                                                                                                | Referencia en `ms-inventory/`                                                  | Qué adaptar                                      |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------ |
| 1   | **Outbox Domain Model** — `OutboxEvent` record + `EventType` enum + `OutboxStatus` enum + `isPending()`/`isPublished()`/`markAsPublished()` + `DomainEventEnvelope` (constante `MS_SOURCE`, compact constructor con defaults)                                             | `domain/model/outboxevent/`                                                    | Valores de `EventType`, valor de `MS_SOURCE`     |
| 2   | **OutboxRelayUseCase** — `BATCH_SIZE=100`, `fetchPendingEvents`, `markAsPublished`                                                                                                                                                                                        | `domain/usecase/outboxrelay/OutboxRelayUseCase.java`                           | —                                                |
| 3   | **ProcessedEvents (idempotencia)** — `ProcessedEventRepository` interface (`exists`, `save`) + `R2dbcProcessedEventAdapter` con `DatabaseClient` INSERT (NO `repository.save()` — UUIDs de fuente externa, §2.2)                                                          | `domain/model/processedevent/gateways/` + `infrastructure/.../processedevent/` | —                                                |
| 4   | **R2dbcOutboxAdapter** — DTO + Mapper + SpringDataRepository                                                                                                                                                                                                              | `infrastructure/driven-adapters/r2dbc-postgresql/.../outbox/`                  | Campos del DTO                                   |
| 5   | **Kafka Outbox Relay** — `KafkaOutboxRelay` (`@Scheduled` externalizado, `onErrorResume()` mantiene PENDING) + `KafkaProducerConfig` (`KafkaSender`, acks=all, retries=3, idempotent)                                                                                     | `infrastructure/driven-adapters/kafka-producer/`                               | Tópico, UseCase inyectado, partition key         |
| 6   | **Kafka Consumer** — `KafkaConsumerConfig` (`KafkaReceiver` por tópico) + `KafkaConsumerLifecycle` (`ApplicationReadyEvent`) + `KafkaEventConsumer` (switch `eventType`, per-msg ack, idempotencia via `processedEventRepository.exists()`, `onErrorResume()` + log WARN) | `infrastructure/entry-points/kafka-consumer/`                                  | Tópico, lógica de procesamiento, deserialización |
| 7   | **Controller → Handler → UseCase** — Controller thin (HTTP + `@Tag`/`@Operation`) → Handler `@Component` (`Mono<ResponseEntity<T>>`) → UseCase + mappers estáticos DTO↔Dominio                                                                                            | `infrastructure/entry-points/reactive-web/.../api/`                            | Entidades, DTOs, rutas                           |

> **§B.12:** `ReactiveKafkaConsumerTemplate` fue eliminado en spring-kafka 4.0 GA. Usar `KafkaReceiver` de `reactor-kafka`.
